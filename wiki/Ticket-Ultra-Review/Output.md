# Output

Findings print to the terminal. With `--comment`, each issue on a line in the PR diff becomes an inline comment, and whole-file, structural, and cross-ticket findings go into one summary comment that leads with the whole-batch run's `N of M tickets ready.` line. An inline comment carries a committable suggestion block only for a small self-contained fix that resolves the issue on its own; larger or multi-location fixes are described without one. One comment is posted per issue. When no issues are found, a single comment records that the tickets were checked. Every posted comment opens with a `🤖 Generated with Claude Code` line.
